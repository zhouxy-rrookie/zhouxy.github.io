并联四足与串联四足：结构路线如何影响控制路线

并联四足与串联四足：结构路线如何影响控制路线

做四足机器人时，很多人一开始关注的是控制算法：

  • 要不要上 MPC？
  • 要不要做强化学习？
  • 需不需要 WBC？
  • 位置控制是不是太“初级”了？

但真正做过系统之后就会发现，很多控制路线并不是“凭空选出来”的。
机械结构本身，往往在一开始就决定了你后面大概率会走哪条控制路线。

四足机器人里，一个非常典型的分叉点就是：

腿部结构到底做并联，还是做串联？

这个问题看起来像纯机械设计，实际上会一路影响：

  • 工作空间与自由度设计
  • 电机与减速器选型
  • 关节输出轴受力方式
  • 腿部惯量与响应速度
  • 位置控制是否足够
  • 力控和柔顺控制是否容易做
  • 步态类型和越障方式
  • 适合走比赛路线，还是高动态 locomotion 路线

所以这篇文章想讨论的，不只是“并联和串联的优缺点”，而是更重要的一个问题：

为什么四足机器人的结构路线，会反过来塑造它的控制路线？

一、先说结论：结构不是控制的前置条件，而是控制的边界条件

很多初学者会下意识地把控制放在“更高层”的位置，觉得：

  • 机械结构先做出来
  • 控制算法后面再补
  • 算法够强，结构差一点也能救

现实通常不是这样。

更准确地说，机械结构是控制系统的物理边界条件。
它决定了控制器面对的是怎样的一台机器：

  • 是高惯量还是低惯量
  • 是大工作空间还是小工作空间
  • 是高扭矩需求还是低扭矩需求
  • 是容易做高带宽响应还是容易做慢速稳态动作
  • 是更适合几何轨迹跟踪，还是更适合动态力控制

因此，结构选型不是“机械组的事”，而是整机路线的第一层决策。

二、什么是串联四足，什么是并联四足

2.1 串联腿：开放链结构

串联腿是四足机器人里最常见、最直观的一类结构。
它的每个关节依次连接，形成一条开放运动链，常见形式是：

  • 髋关节 abduction/adduction
  • 髋关节 flexion/extension
  • 膝关节 flexion/extension

也就是我们经常见到的 3 自由度串联腿。

这种结构的直觉优势很明显：

  • 结构表达直观
  • 自由度丰富
  • 工作空间大
  • 容易做复杂落脚点调整
  • 更接近现代高机动四足的主流形态

很多经典四足、商业四足和高动态研究平台，都是这类路线。

2.2 并联腿：闭链结构

并联腿则是另一种思路。
它的腿部通常由多根杆件构成闭环，多个支链共同决定足端位置。
在一些竞赛型和轻量化设计中，常见的是 2 自由度并联腿，电机尽量布置在机体附近，通过连杆带动腿部运动。

相关研究中明确指出，并联腿部构成封闭运动链，能够把驱动电机布置在机体，从而减小腿部运动惯量，提高动态性能和响应速度。fileciteturn5file1L5-L6

三、串联与并联的第一层差异：不是“谁高级”，而是“谁更像你要的机器”

关于这两种结构，最常见的误区就是问：

哪种结构更先进？

这其实是个不太好的问题。
更好的问题应该是：

你想做的是一台什么样的四足？

因为并联和串联并不是简单的“替代关系”，而更像是两种不同的系统取向。

3.1 串联腿更像“通用型四足平台”

串联腿通常有更大的工作空间，正运动学求解也更直接。相关论文总结串联构型的优点包括：工作空间大、正运动学求解容易、结构简单、使用方便。fileciteturn5file1L13-L15

这意味着它更适合：

  • 复杂三维足端运动
  • 精细落脚点控制
  • 非结构化地形适应
  • 更高自由度的全身运动
  • WBC / MPC / RL 等高维控制策略

3.2 并联腿更像“性能集中型、任务导向型平台”

并联腿虽然通常工作空间更小、正运动学更复杂，但它在另一些指标上很有优势。研究总结并联腿的优点包括：不易产生累积误差、几何误差具有平均化效果、更容易达到高精度；可将驱动电机布置在机体，腿部惯量更小，灵活性和响应速度更高；承载杆件更多，承载能力更强、结构更稳定。fileciteturn5file1L15-L18

所以并联腿往往更适合：

  • 轻量化竞速
  • 结构化赛道
  • 以跳跃、快速越障为核心的任务
  • 对成本和制造复杂度敏感的项目
  • 电机能力有限但又希望“先把东西跑起来”的平台

四、机械结构如何改变控制问题：先从“扭矩需求”开始

很多控制路线之所以分叉，并不是因为算法理念不同，而是因为扭矩预算根本不一样。

一份很有代表性的比赛经验分享里提到：在大多数情况下，用 trot 步态进行力矩计算是合适的，此时按双足支撑并预留冗余；“显而易见，采用并联腿设计所需的力矩明显小于串联腿设计”。同一份材料还给出过串联腿的实际失败案例：由于力臂过长，小腿关节电机扭矩不足，机器人只能勉强站立，一旦开始行走，后腿就会弯曲倒地，无法维持支撑。fileciteturn5file2L44-L47

这段经验其实非常关键，因为它说明了一个工程上的事实：

结构路线会直接决定你的电机有没有资格去谈控制路线。

4.1 为什么串联腿更容易把扭矩问题暴露出来

串联腿的问题不一定是“理论上不行”，而是：

  • 力臂往往更长
  • 下游关节受力更直接
  • 关节电机更容易成为瓶颈
  • 腿部杆件和执行器一起运动，惯量更大
  • 高动态时对峰值扭矩和功率密度要求更高

如果电机选型、减速比、散热、传动效率不够，控制器还没来得及发挥，硬件就先撑不住了。

4.2 为什么并联腿在很多低成本项目里更“友好”

并联腿通过机构关系降低了部分关节直接承受的大力矩需求，同时电机更容易布置在机体，进一步减小摆腿惯量。相关研究也明确指出，并联腿由于驱动电机可放在机体、输出轴主要承受轴向力，整体承载能力更强，结构更稳定。fileciteturn5file1L16-L18

这会带来两个直接结果：

  1. 用相对普通的电机，也更有机会先把机器人做成
  2. 位置控制 + 轨迹规划 这类路线更容易先跑通

所以很多竞赛型四足不是“因为不知道先进控制”，而是因为在那个硬件条件下，这就是更合理的结构-控制组合。

五、并联腿为什么天然更容易走“位置控制优先”的路线

这一点在比赛型四足里尤其明显。

在大连工业大学的技术分享中，有一句非常有代表性的话：
**8 自由度并联四足在 RoboCon 场景里很受欢迎，最主要的原因之一就是结构简单、成本较低；另一个重要原因是平台控制相对简单，采用位置控制就可以实现跳跃和稳定运动，同时比赛地形主要以结构化为主。**fileciteturn5file0

这句话背后，其实是一个完整的逻辑链：

  • 并联腿通常更轻、更紧凑
  • 任务地形结构化，落脚点和动作模式可预先设计
  • 需要的是稳定、重复、快速实现的动作
  • 于是控制上更偏向：
    • 逆运动学
    • 足端轨迹规划
    • 位置闭环
    • 状态机切换
    • 少量姿态补偿

5.1 为什么这种路线在比赛里很合理

比赛场景和野外场景不同。
如果你的任务是：

  • 跑固定路线
  • 跳过明确高度的障碍
  • 在已知坡度和台阶上通过
  • 追求开发效率和稳定复现

那么“能稳定实现”的位置控制，往往比“理论上更先进”的复杂控制更有价值。

一项关于并联四足跳跃的研究就是典型例子：其方法是先规划机体质心轨迹，再通过逆运动学求关节角，最后使用位置控制实现跳跃，同时引入柔顺设计降低落地冲击。fileciteturn4file11

也就是说，并联腿常常天然导向一种控制哲学：

先把几何关系和轨迹规划做好，再用位置控制把动作稳定打出来。

六、串联腿为什么更容易走向“力控、全身控制和学习控制”

与并联腿相对，串联腿通常更容易引出另一条路线：

  • 更强的关节可控性
  • 更自然的三维足端调节
  • 更适合复杂地形和接触变化
  • 更适合统一放进动力学控制框架

6.1 串联腿更适合复杂地形和高维任务

从结构直觉上讲，串联腿更接近“通用肢体”的感觉。
因为工作空间更大，腿部动作更自由，对地形和障碍的适应手段也更多。

这使得串联腿更容易承接下面这些控制路线：

  • 足端力控制
  • 阻抗控制
  • WBC
  • MPC
  • 视觉 locomotion
  • RL / residual RL / AMP

6.2 为什么高动态四足大多是串联路线

这并不是因为并联腿不能做动态运动，而是因为串联结构在自由度配置和动作表达上更接近现代高动态控制的需求。

学习型足式综述提到，近年来四足硬件快速发展，一个核心原因是高扭矩密度、低减速比、可回驱执行器的发展，使得直接关节电流控制、高带宽力控制和更强的抗冲击能力成为可能。fileciteturn5file3L2-L2

这类执行器与串联腿结合后，才能真正把以下路线推起来：

  • 支撑腿力分配
  • 质心与角动量控制
  • 接触约束下的优化控制
  • 模型预测控制
  • 学习策略的高频执行

所以串联腿不只是“结构不同”，而是更容易进入现代动力学控制和学习控制的话语体系。

七、运动学与动力学难度，也会改变控制路线

结构不同，数学模型的重心也不同。

7.1 串联腿：正运动学简单，逆运动学直观，但累计误差和惯量问题突出

相关研究指出，串联腿虽然正运动学求解容易、结构简单，但也存在几个典型问题：

  • 每个关节独立驱动，容易产生累计误差
  • 不能只靠编码器，需要额外矫正
  • 承载能力相对差
  • 驱动电机常布置在杆件上，运动惯量大，影响灵活性
  • 逆运动学求解也会随着自由度和构型增加而更复杂。fileciteturn5file1L13-L16

这些问题意味着：
串联腿虽然控制自由度高，但你必须投入更多精力处理：

  • 标定
  • 惯量补偿
  • 动力学建模
  • 力矩预算
  • 状态估计
  • 接触控制

7.2 并联腿：几何精度和轻量化很强，但模型更像“专用机器”

并联腿的另一面是：

  • 正运动学常更复杂
  • 工作空间受限
  • 足端可达区域和姿态空间更受约束
  • 结构对几何尺寸、连杆比例更敏感

但它的优势是几何误差不易累积，腿部更轻，速度响应快。研究中对并联腿还专门做了连杆长度优化、工作空间分析，并把这些作为步态规划和控制优化的基础。fileciteturn5file1L5-L6

这说明并联腿常常更适合一种“先把结构调到很顺手，再围绕它量身定做控制器”的路线。

八、步态和越障方式，也会被结构深刻影响

步态从来不是控制器单方面决定的，结构会直接改变“这台机器人擅长怎么走”。

8.1 并联四足更偏“跳、冲、过结构化障碍”

在赛事经验里，有一个很直白的判断：
结构化地形其实对 8 自由度并联四足很友好，因为“毕竟没有什么障碍是不能跳过去的”。材料中还提到，对于高墙这类障碍，串联四足常采用前后腿“扒拉”方案，存在一定失误率。fileciteturn5file2L39-L40

这句话虽然很口语，但非常准确地点出了并联路线的长处：

  • 机体轻
  • 腿惯量小
  • 爆发式动作容易做
  • 在固定赛道里，跳跃往往比精细接触规划更直接

8.2 串联四足更偏“走法丰富、地形适应更细”

串联腿因为工作空间大、落脚可调范围广，更适合做：

  • 精细落脚点调整
  • 抬腿避障
  • 复杂姿态切换
  • 不规则地形探测式步行
  • 依赖足端受力和状态估计的动态步态

因此你会看到一个很常见的现象：

  • 并联路线常在结构化地形和比赛中打得很漂亮
  • 串联路线更容易在“通用 locomotion”语境里成为主流

九、执行器路线如何和结构路线绑定

四足机器人的控制上限，不是由控制器单独决定的，而是由“结构 + 执行器 + 控制”共同决定的。

9.1 高减速比、不可回驱，不利于高带宽力控

学习型足式综述里提到，传统高减速比方案会引入较大的静摩擦，使关节不可回驱，难以通过简单电流控制实现输出力矩控制；后来正是低减速比、高扭矩密度、可回驱执行器推动了四足 locomotion 的快速发展。fileciteturn5file3L2-L2

这意味着：

  • 如果你的结构路线天然更依赖高减速、慢响应、低带宽，那么控制路线大概率就会更偏位置控制
  • 如果你的结构路线能配合可回驱、高带宽执行器，那你才更有机会真正把力控、MPC、RL 做起来

9.2 结构设计甚至会影响电机安装方式和输出轴安全性

在赛事经验中，还有一些非常细节但极重要的提醒，例如：

  • 不要让螺丝直接承担关节输出轴的主受力
  • 碳管固定方式如果设计不好，整机会扭成平行四边形
  • 足端材料和缓冲结构会直接影响落地稳定性和噪声表现。fileciteturn5file2L40-L49

这些问题看似“只是机械细节”，但它们实际决定了控制器能否在真实世界里按预期工作。

十、为什么很多人觉得“串联更先进、并联更比赛化”

这个印象不是完全没道理，但也不能简单化。

10.1 这个印象为什么会出现

因为今天主流公开可见的高动态四足、商业四足、学习型四足，大多数采用的是串联腿路线。
这会给人一种印象：串联是“现代路线”，并联像是“早期路线”或“比赛特化路线”。

10.2 但并联并不等于落后

并联路线并不是技术落后，而是问题定义不同。
例如在一篇并联四足论文中，作者明确采用 2 自由度轻量级并联腿，并围绕它建立了工作空间分析、SLIP 建模、VMC 控制、多步态实验和越障实验，验证了该路线在高速跑动、跳跃和越障方面的有效性。fileciteturn5file1L5-L10

所以更准确的理解是：

  • 串联路线更适合追求通用性和上限
  • 并联路线更适合在受限成本和受限任务下追求高性价比性能

十一、从控制角度看：并联和串联分别容易走向什么技术栈

这部分最适合直接总结。

11.1 并联四足常见控制路线

并联四足更容易走向下面这套技术栈：

  1. 单腿逆运动学
  2. 足端几何轨迹规划
  3. 关节位置控制 / PD
  4. 步态状态机
  5. 少量姿态补偿
  6. 跳跃、越障、快速动作模板
  7. 在需要时再加入简化模型控制

这种路线的特点是：

  • 上手快
  • 调试效率高
  • 对结构化任务非常有效
  • 对硬件要求相对友好

11.2 串联四足常见控制路线

串联四足则更容易走向：

  1. 完整运动学与动力学建模
  2. 状态估计
  3. 足端力 / 关节力矩控制
  4. 阻抗控制
  5. WBC / QP 力分配
  6. MPC
  7. RL / residual RL / AMP / 视觉 locomotion

这种路线的特点是：

  • 泛化能力更强
  • 更适合复杂地形和动态任务
  • 算法空间更大
  • 但对执行器、计算平台和系统集成要求更高

十二、如果我要做项目，应该怎么选

这个问题没有统一答案，但可以按目标来判断。

12.1 更适合优先考虑并联腿的情况

如果你的项目目标是：

  • 先做出能跑能跳的样机
  • 预算有限
  • 比赛场景较结构化
  • 电机能力有限
  • 团队更擅长机械与嵌入式
  • 希望优先走位置控制和轨迹规划路线

那并联腿通常是很有吸引力的方案。

12.2 更适合优先考虑串联腿的情况

如果你的项目目标是：

  • 做更通用的 locomotion 平台
  • 走复杂地形适应路线
  • 关注 WBC / MPC / RL 等先进控制
  • 使用关节电机和高带宽执行器
  • 希望后续接视觉、感知和学习策略
  • 希望平台具备长期扩展性

那串联腿通常会更合适。

十三、一个更成熟的工程判断：先问任务，再定结构，再谈控制

很多讨论容易本末倒置，一上来就问：

  • 这篇论文里用的是不是 MPC？
  • 这台机器人有没有上强化学习？
  • 位置控制是不是太 low？

其实真正成熟的工程顺序应该是：

  1. 任务是什么
    是比赛？巡检？科研平台？越障展示？高动态跑跳？还是学习型平台？

  2. 结构选什么
    并联还是串联？几自由度？工作空间和扭矩预算怎样？

  3. 执行器能到什么水平
    能不能做高带宽力控？是否可回驱？减速比和峰值扭矩够不够？

  4. 控制该怎么配
    位置控制、阻抗控制、VMC、WBC、MPC、RL 分别是否匹配当前硬件条件？

这四步里，结构处在非常靠前的位置。
所以四足机器人里常说“机械决定下限，控制决定上限”，这句话其实还可以再改一改：

机械结构决定了你到底有没有资格去谈某种控制上限。

十四、总结

并联四足与串联四足的差异，从来不只是机械构型差异。
它们实际上代表了两条不同的系统路线。

串联腿通常意味着：

  • 更大的工作空间
  • 更丰富的动作表达
  • 更适合复杂地形和高维控制
  • 更容易走向力控、WBC、MPC 和 RL

并联腿通常意味着：

  • 更低的腿部惯量
  • 更强的承载与结构稳定性
  • 更容易在有限硬件条件下做出高性能样机
  • 更适合结构化场景、跳跃越障和位置控制优先路线

所以，真正值得问的问题不是：

并联和串联谁更先进？

而是：

在我的任务目标、预算、电机能力、团队经验和开发周期之下，哪一条结构路线更能支撑我想走的控制路线？

这才是一个四足机器人项目真正的起点。
